# סכמת נתונים - nb_clinic_content_dashboard_3.html

כל הנתונים חיים בתוך שלושה מערכי JS גלובליים, בתוך תג `<script>` אחד בקובץ.
אין build step, אין תלות חיצונית - עורכים את הקובץ ישירות.

## שלושת המערכים

### `posts` - פוסטים משובצים לתקופה הנוכחית

פוסטים עם `day` מספרי (1, 2, 3...) שמייצג יום עבודה בתוך התקופה המוצגת
כרגע בכותרת (`.top-sub`). ה-`day` **לא** תאריך קלנדרי - הוא אינדקס יחסי
לתחילת התקופה (רצף ימי א'-ה', בלי סופ"שים, לפי הדפוס הקיים).

### `funnelPosts` - פוסטים למשפך השקה פעיל

במקום `day`, יש להם `stage` (1-5) ו-`stageLabel` (למשל `"שלב 1 — מודעות"`).
משפך אינו קשור לתקופת זמן ספציפית - הוא נשאר קבוע לאורך כל ההשקה, ללא קשר
לכמה מחזורי `posts` עוברים באמצע.

### `poolPosts` - ספריית רעיונות אורגניים (לא משובצים)

אותה סכמה בדיוק, אבל `day: null`. אלה רעיונות מוכנים (כולל תסריט וקפשן
מלאים, או לפעמים רק תיאור בלי תסריט אם עוד לא נכתב) שממתינים לשיבוץ עתידי
או שכבר "סיימו את התפקיד" בתקופה קודמת ונשמרים לשימוש חוזר.

## צורת אובייקט פוסט (זהה בשלושת המערכים, חוץ מ-day/stage)

```js
{
  id: 34,                     // ייחודי בכל שלושת המערכים יחד, אף פעם לא לעשות שימוש חוזר
  day: 1,                     // או null (pool) / להחליף ב-stage+stageLabel (funnel)
  topic: "כותרת קצרה וממוקדת של הפוסט",
  format: "ריל",              // "ריל" | "קרוסל" | "פוסט סטטי" | "סטורי" - בדיוק כך, כולל רווח ב"פוסט סטטי"
  audience: "הריון",          // "הריון" | "אחרי לידה" | "לפני היריון"
  goal: "חשיפה",              // טקסט חופשי קצר, למשל "ערך + ליד", "אמפתיה + מכירה"
  desc: "תיאור פנימי לנועם - על מה הפוסט ולמה, לא הקפשן עצמו",
  cta: "שמרי את זה / הורידי MomSafe",
  source: "רעיון חדש - תכנון <תאריך/הקשר>",
  isNew: true,                // true לכל פוסט שנוסף במחזור הזה
  hasScript: true,            // false אם עדיין אין תסריט מלא (כמו פרקי המשך בסדרה)
  caution: "לנסח כ'קשור למחקרים מסוימים'...",  // אופציונלי - רק כשיש רגישות מדעית/רפואית
  script: `...`,              // template string מלא, או null אם hasScript=false
  caption: `...`,             // template string מלא, או null (למשל בסטורי-פול שאין לו קפשן נפרד)
  hashtags: "#... #NBClinic", // string אחד, או null
}
```

שדות שאפשר להשמיט אם אין להם ערך: `caution`. שאר השדות תמיד קיימים (עם
`null` כערך כשרלוונטי, כמו ב-`caption: null, hashtags: null` לפוסטי סטורי).

## מבנה סקריפט לפי פורמט

**ריל** - שלושה בלוקים עם תזמון בשניות:
```
HOOK (0–3 שניות):
"..."

BODY (3–XX שניות):
...

CLOSE + CTA (XX–YY שניות):
...
```

**קרוסל** - שקף-שקף, ממוספר, כל שקף שורה-שתיים:
```
שקף 1 (פתיחה):
...

שקף 2:
...

שקף N (CTA):
...
```

**סטורי** - תלוי בסוג (poll/quiz/רצף), לדוגמה:
```
[סטורי — פורמט Poll/Quiz]

כותרת עליונה:
"..."

אפשרויות (Poll עם 3-4 כפתורים):
...
```

## מבנה קפשן (לפוסטים עם caption)

הוק (שורה ראשונה שמושכת קריאה - לא "היי!") → גוף (ערך/סיפור, שורות קצרות)
→ חיבור אישי → CTA אחד ברור. בלי שורת האשטגים בתוך ה-caption עצמו -
ה-hashtags נשמרים בשדה נפרד.

## נוהל עדכון בלתי-הרסני (לבצע בכל מחזור חדש)

זו הפעולה הכי חשובה בסקיל הזה - **אף פעם לא לדרוס/למחוק תוכן קיים**, גם אם
התקופה שלו חלפה. ה-id-ים מקושרים ל-state בלוקאלסטורג' של הדפדפן (שיבוץ
ליומן ב-`nb_marketing_calendar`, משימות ב-`nb_marketing_tasks`) - מחיקה או
שינוי id שובר את זה בשקט.

צעדים מדויקים:

1. **למצוא את ה-id המקסימלי** הקיים כרגע בכל שלושת המערכים יחד (`posts`,
   `funnelPosts`, `poolPosts`) - ה-id-ים החדשים ימשיכו מ-1 מעל זה.
2. **להעביר פוסטים ישנים ל-pool**: כל אובייקט ב-`posts` עם `day` לא-null
   ששייך לתקופה שמסתיימת - להעתיק אותו (בלי לשנות שום שדה חוץ מ-`day`)
   לתוך `poolPosts`, ולשנות בו `day: null`. להוציא אותו מ-`posts`.
3. **להוסיף את הפוסטים החדשים** ל-`posts` עם `day: 1..N` לפי סדר הפרסום
   המתוכנן בתקופה החדשה.
4. **לעדכן את הכותרת הסטטית** ב-HTML (לא ב-JS): `.top-sub` (טווח תאריכים),
   `.top-meta` (הפילים - מוצרים במיקוד), `.focus-banner-text` (משפט הפוקוס).
5. **`funnelPosts` נשאר כפי שהוא** אלא אם המשתמשת מבקשת במפורש להוסיף שלב
   או פוסט חדש למשפך - ואז רק מוסיפים, לא מוחקים.
6. **לא לגעת** בשום קוד מעבר לשלושת המערכים והכותרת הסטטית - כל שאר ה-JS
   (רינדור, `LS_CAL`, `LS_TASKS`, מודל, לוח שנה) כבר עובד.
