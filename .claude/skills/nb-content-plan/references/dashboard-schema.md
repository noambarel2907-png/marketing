# סכמת נתונים - nb_clinic_content_dashboard_3.html

עודכן: יולי 2026, אחרי שדרוג משמעותי לדשבורד מבוסס טאבים. כל הנתונים חיים
בתוך כמה מערכי JS גלובליים, בתוך תג `<script>` אחד בקובץ. אין build step,
אין תלות חיצונית - עורכים את הקובץ ישירות.

## מבנה הטאבים (לא לגעת בלוגיקת הרינדור/הטאבים עצמם)

| טאב | תוכן |
|---|---|
| 🚀 פאנלים | תרשימי זרימה של נקודות כניסה עסקיות (`ENTRY_PANELS`) - שינוי אסטרטגי, לא שגרת תוכן |
| 🗓️ אסטרטגיית תוכן | לוח שנה חודשי, **ריק כברירת מחדל** - המשתמשת משבצת אליו בעצמה מתוך "פוסטים אורגניים" |
| ✅ משימות לביצוע | משימות (אוטומטיות + ידניות), ניתנות למחיקה |
| 🎬 תסריטים לצילום | כל הסקריפטים המוכנים + תת-כותרת קבועה "יום צילום עם מאיה ועופרי" (`MAYA_OFRI_DAY1`, ארכיון קפוא) |
| 🤖 רעיונות מקלוד | **כאן הסקיל הזה כותב** - `CLAUDE_IDEAS`, ראו למטה |
| 🌱 פוסטים אורגניים | ספריית התוכן המאושר, מקובצת לפי תיקיית מטרה (`CATEGORIES`) |

**אסור לסקיל הזה לגעת ב:** `ENTRY_PANELS`, `CATEGORIES`, מבנה הטאבים (`TABS`),
לוגיקת הרינדור/הקלנדר/המודל. אלה משתנים רק ביוזמת המשתמשת במפורש בבקשה
נפרדת, לא כחלק משגרת תכנון תוכן.

## מערכי הפוסטים

### `CLAUDE_IDEAS` - **היעד היחיד לכתיבה של הסקיל הזה**

מערך שמתחיל ריק ומצטבר עם הזמן - כל הרצה של הסקיל **מוסיפה** רעיונות חדשים
לסופו, **אף פעם לא מוחקת/דורסת** מה שכבר שם. המשתמשת רואה את הרעיונות
בטאב "🤖 רעיונות מקלוד", שם היא יכולה לפתוח כל כרטיס לעריכה מלאה (כותרת/
סקריפט/קפשן), למחוק רעיון שלא מתאים, או ללחוץ "➡️ העברה לפוסטים אורגניים" -
פעולה שהדשבורד עצמו מבצע (לא הסקיל!): היא מוסיפה שדות `persona`/`priority`
ומסווגת אוטומטית לקטגוריה (`inferCategory()` לפי מילות מפתח ב-goal/cta/desc)
לפני שהפוסט מופיע בטאב "פוסטים אורגניים".

**המשמעות עבור הסקיל:** לעולם לא לגעת ב-`ORGANIC_POOL` או ב-`CALENDAR_POSTS`
ישירות - ה"מעבר" לספרייה הראשית הוא פעולת משתמשת בדשבורד, לא פעולת כתיבה
של הסקיל. הסקיל כותב אך ורק ל-`CLAUDE_IDEAS`.

### `CALENDAR_POSTS` ו-`ORGANIC_POOL` - קפואים, לא לגעת

אלה הפוסטים ההיסטוריים/המאושרים שכבר קיימים בדשבורד (כולל אלה שהועברו
בעבר מ"רעיונות מקלוד"). הם ממוזגים יחד בזמן ריצה לתוך `LIBRARY_POSTS`.
הסקיל הזה לא כותב ולא עורך אותם - הם באחריות המשתמשת/הדשבורד בלבד.

### `MAYA_OFRI_DAY1` - ארכיון קפוא, לא לגעת

תסריטי הפקה מדויקים ממסמך קריאייטיב חיצוני (יום צילום עם מאיה ועופרי).
לא קשור לרעיונות תוכן שוטפים - הסקיל הזה לא נוגע בו בכלל.

## צורת אובייקט ב-`CLAUDE_IDEAS`

```js
{
  id: 61,                     // ייחודי במכלול (ראו "מציאת ה-id הבא" למטה), אף פעם לא לעשות שימוש חוזר
  topic: "כותרת קצרה וממוקדת של הפוסט",
  format: "ריל",              // "ריל" | "קרוסל" | "פוסט סטטי" | "סטורי" - בדיוק כך, כולל רווח ב"פוסט סטטי"
  audience: "הריון",          // "הריון" | "אחרי לידה" | "לפני היריון" | "שניהם"
  goal: "חשיפה",              // טקסט חופשי קצר, למשל "ערך + ליד", "אמפתיה + מכירה" - חשוב: זה חלק מהקלט
                               // ל-inferCategory() כשהמשתמשת תעביר את הפוסט לספרייה, אז שיהיה מדויק
  desc: "תיאור פנימי לנועם - על מה הפוסט ולמה, לא הקפשן עצמו",
  cta: "שמרי את זה / הורידי MomSafe",
  source: "רעיון קלוד — <תיאור קצר של הבקשה/הקשר> (<תאריך>)",
  isNew: true,
  hasScript: true,            // false אם עדיין אין תסריט מלא
  caution: "לנסח כ'קשור למחקרים מסוימים'...",  // אופציונלי - רק כשיש רגישות מדעית/רפואית
  script: `...`,              // template string מלא, או null אם hasScript=false
  caption: `...`,             // template string מלא, או null
  hashtags: "#... #NBClinic", // string אחד, או null
}
```

**אין שדה `day`** - שיבוץ ליומן קורה אך ורק דרך הדשבורד עצמו (טאב
"אסטרטגיית תוכן"), אחרי שהמשתמשת מעבירה את הפוסט ל"פוסטים אורגניים".
הסקיל הזה **לעולם לא** מציע תזמון/day - זה בכוונה בידיה של המשתמשת בלבד.

## מציאת ה-id הבא (חובה בכל הרצה)

לפני הוספת רעיונות חדשים, למצוא את ה-id המקסימלי הקיים כרגע ב-**ארבעת**
המערכים יחד: `CALENDAR_POSTS`, `ORGANIC_POOL`, `MAYA_OFRI_DAY1`, ו-`CLAUDE_IDEAS`.
ה-id-ים החדשים ממשיכים מ-1 מעל הגבוה מכולם. אל תסתמכי על מונה נפרד לכל
מערך - יש state בלוקאלסטורג' של הדפדפן (`nb_marketing_calendar`,
`nb_manual_tasks`, `nb_transferred_claude_ideas`, `nb_deleted_claude_ideas`,
`nb_post_edits`) שמפנה לפוסטים לפי id בין כל המערכים - התנגשות id שוברת
את זה בשקט.

## מבנה סקריפט לפי פורמט

**ריל** - שלושה בלוקים עם תזמון בשניות:
```
HOOK (0–3 שניות):
"..."

BODY (3–XX שניות):
...

CLOSE + CTA (XX–YY שניות):
...
```

**קרוסל** - שקף-שקף, ממוספר, כל שקף שורה-שתיים:
```
שקף 1 (פתיחה):
...

שקף 2:
...

שקף N (CTA):
...
```

**סטורי** - תלוי בסוג (poll/quiz/רצף), לדוגמה:
```
[סטורי — פורמט Poll/Quiz]

כותרת עליונה:
"..."

אפשרויות (Poll עם 3-4 כפתורים):
...
```

## מבנה קפשן (לפוסטים עם caption)

הוק (שורה ראשונה שמושכת קריאה - לא "היי!") → גוף (ערך/סיפור, שורות קצרות)
→ חיבור אישי → CTA אחד ברור. בלי שורת האשטגים בתוך ה-caption עצמו -
ה-hashtags נשמרים בשדה נפרד.

## נוהל עדכון בלתי-הרסני (לבצע בכל הרצה)

1. **למצוא את ה-id המקסימלי** בארבעת המערכים יחד (ראו למעלה).
2. **להוסיף אך ורק ל-`CLAUDE_IDEAS`** - לאתר את הסוגריים הסוגרים `];` של
   המערך הזה, ולהוסיף את האובייקטים החדשים לפניהם. **לא לגעת** ב-
   `CALENDAR_POSTS`, `ORGANIC_POOL`, או `MAYA_OFRI_DAY1`.
3. **לא להוסיף `day`** לאף פוסט - שיבוץ הוא אך ורק פעולה ידנית של המשתמשת
   בדשבורד עצמו.
4. **`source`** תמיד מתאר את הקשר ההרצה הזו (מה ביקשה נועם, תאריך), כדי
   שיהיה ברור בטאב "רעיונות מקלוד" מאיפה כל רעיון הגיע.
5. **לא לגעת** בשום קוד/מערך אחר - `ENTRY_PANELS`, `CATEGORIES`, `TABS`,
   `CALENDAR_POSTS`, `ORGANIC_POOL`, `MAYA_OFRI_DAY1`, הכותרת הסטטית
   (`.top-sub`/`.top-meta`/`.focus-banner-text`), או כל לוגיקת רינדור/
   קלנדר/משימות/מודל - כל זה כבר עובד ומנוהל בנפרד על ידי המשתמשת.
